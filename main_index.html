한국교통안전공단은 2017~2019년 최근 3년간 개인형 이동수단(PM-Personal Mobility)으로 인한 교통사고 분석 결과를 발표했습니다. 2017년 부상자 수 는 124명, 2018년은 238명, 2019년은 473명으로 연평균 사고건수와 부상자수가 95%이상 증가하고 있습니다. 사고율이 증가한 만큼 저희는 주행자의 안전과 골든타임을 지키기 위해 딥러닝 가빈 퍼스널 모빌리티 낙상 감지 시스템을 구상했습니다.

2. 구동 원리

낙상 데이터와 주행 데이터를 수집하여 인공신경망(Convolutional Neural Network, CNN)을 통해서 주행 중인 상황인지 낙상 상황인지 판별합니다. 주행 및 낙상 데이터는 3축 가속도 센서 데이터 값으로 수집하며 데이터 수집 밥법은 주행 데이터 수집 전용 H/W를 통해서 수집하고 낙상 데이터 수집도 마찬가지로 낙상 데이터 수집 H/W를 통해서 수집합니다. 모은 데이터를 인공신경망 즉, CNN을 통해서 훈련하고 출력되는 모델에 3축 가속도 센서 데이터 값을 입력하면 낙상을 감지할 수 있습니다.

3. H/W

CNN 기반 낙상 감지 시스템에 사용되는 하드웨어는 아두이노 우노 보드, 3축 가속도 센서, 잿슨 나노가 있습니다.

3축 가속도 센서는 낙상 여부와 주행 상태를 파악하기 위한 중요한 센서로 x, y, z 축을 측정하는 센서입니다. 뿐만 아니라 3축 자이로 센서는 낙상 및 주행 데이터 수집하기 위해서도 사용됩니다.

위의 사진에 보이는 아두이노 우노 보드는 마이크로컨트롤러(Microcontroller)로 44개의 핀과 단자를 통해서 다른 보드 또는 센서들의 제어할 수 있어서 3축 가속도 센서 데이터를 읽어들이는 역할을 합니다.

잿슨나노의 역할은 CNN 기반 낙상 감지 모델을 탑재하고 있으며 아두이노 우노에서 송신한 3축 가속도 센서 데이터 깂을 수신하여 CNN 기반 낙상 감지 모델에 입력하여 사용자의 시스템 전용 앱에 낙상 감지 여부 데이터를 송신합니다.

3. 통신

아두이노 우노 보드에서 3축 가속도 센서를 CNN 기반 낙상 감지 모델에 송신하기 위해 USB 시리얼 통신을 사용합니다. USB 시리얼 통신은 위의 사진처럼 USB 포트를 통해서 데이터를 송/수신하는 방법으로 간단한 인터페이스로 통신이 가능합니다.

Wi-Fi 다이렉트 통신은 잿슨 나노에 탐재된 CNN 기반 낙상 감지 모델에서 낙상이 판별될 경우 사용자의 휴폰에 설치된 전용 앵에 닉싱 데이터를 송신하기 위해서 사용됩니다. 사용자의 전용 앱에서 낙상 데이터를 받게 되면 자동 신고 프로세스가 진행되어 지인이나 119에 위치 정보가 담긴 SMS 송신이 이루어지게 됩니다.

Deep Rurnning

MFCC/CNN

1.Librosa 라이브러리에서 MFCC를 사용하여 음성데이터를 2D이미지화
2.이미지화 시킨 데이터를 CNN을 통해 DeepRurnning
-이미지 input
-2개의 'Conv2D' 컨볼루션 레이어로 구성된 간단한 모델 아키텍처로 시작하는 '순차' 모델을 사용
-최종 출력 레이어는 'dense' 레이어
-컨볼루션 레이어는 특징 감지를 위해 설계.
-필터 창을 입력 위로 밀고 행렬 곱셈을 수행하고 그 결과를 기능 맵에 저장하는 방식으로 작동
-'filter' 매개변수는 각 레이어의 노드 수를 지정
-각 레이어는 16, 32, 64에서 128로 크기가 증가하는 반면 `kernel_size` 매개변수는 커널 창의 크기를 지정하며 이 경우 2는 2x2 필터 행렬
-첫 번째 레이어는 (40, 174, 1)의 입력 모양을 수신합니다. (40은 MFCC, 174은 프레임 수, 1은 오디오가 모노)
-컨볼루션 레이어에 사용할 활성화 함수는 'ReLU'
-20%의 'Dropout' 값을 사용(오버피팅 방지)
-'MaxPooling2D' 유형의 풀링 레이어와 'GlobalAveragePooling2D' 유형의 최종 컨볼루션 레이어가 연결
(풀링 계층은 훈련 시간을 단축하고 과적합을 줄이는 역할)
-Max Pooling유형은 각 창의 최대 크기를 사용하고 GlobalAveragePooling 유형은 'dense'출력 레이어에 공급하기에 적합한 평균을 취합
-출력 레이어에는 가능한 분류 수와 일치하는 10개의 노드(num_labels), 출력 레이어에 대한 활성화는 'softmax'
-Softmax는 출력 합을 최대 1로 만들어 출력을 확률로 해석
-모델은 확률이 가장 높은 옵션을 기반으로 예측

Android Studio

Driving Principle

Main Thread: 활동 시작,생성,시작,재개 -> 활동이 실행 중
Thread : sound input -> 오디오 녹음-> tensorflow에 작업 보내기
Thread : Tensorflow를 통해 오디오 분석 ->tensorflow 모델을 기반으로 사운드 정의(위험음 판단) -훈련 pb 파일 =>Main Thread에 전달
Main Thread: Output -> 진동 And 디스플레이

execution

앱실행-> 대기화면 -> 사운드감지(차량 경적, 사이렌) -> 진동 과 디스플레이로 경고!
